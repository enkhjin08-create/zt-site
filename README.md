# Зөвхөн түүнд — шинэ вэбсайт (Netlify хувилбар)

`zuvhuntuund.com`-ийн агуулахаас дахин татсан бодит өгөгдөл дээр суурилсан, **албан ёсны brand
guideline-ийн дагуу** (өнгө, лого, фонт) бүтсэн, бэлгийн хайрцаг бэлдэх interactive builder-тэй,
**жинхэнэ нууцлалтай** Админ хэсгийг агуулсан шинэ сайт.

## 1. Дотор юу байна

```
index.html      → Нүүр хуудас (hero, "Хэнд бэлэглэх вэ?" gift-tag навигаци, ангиллын товч харагдац)
category.html   → Бүтээгдэхүүн (бүх 103 бараа, ангилал + хүлээн авагчаар шүүх боломжтой)
product.html    → Барааны дэлгэрэнгүй (?id=... параметрээр)
builder.html    → Бэлэг бэлдэх — 8 алхамт interactive configurator (Хэн → Сав → Гол бэлэг → Нэмэлт → Зурвас → Цэцэг → Мэндчилгээ → Дуусгах)
admin.html      → Админ хэсэг — орж ирсэн захиалгуудыг харах, төлөв шинэчлэх (PIN-тэй)
netlify.toml       → Netlify-д Function хаана байгааг хэлдэг тохиргоо
netlify/functions/orders.js → Серверийн код (нууц key, PIN бүгд ЗӨВХӨН энд)
assets/data.js     → Бараа, ангилал, хүлээн авагчийн өгөгдөл (энд засна!)
assets/app.js      → Нийтлэг логик (хайрцагны state, icon-ууд, header/footer)
assets/orders.js   → Клиент талын туслах функцууд — Function-той ярьдаг, нууц зүйл агуулахгүй
assets/style.css   → Дизайн систем (brand guideline-аас)
assets/images/     → Лого (icon + wordmark), PNG хэлбэрээр
assets/fonts/      → Museo Sans Cyrillic — бодит brand фонт (.woff2)
```

Хайрцаг сонгох мэдээлэл хэрэглэгчийн **browser localStorage**-д хадгалагдана.

## 2. Brand guideline-аас орсон өөрчлөлтүүд

Танай `ZuvkhunTuund_MiniGuideline.pdf`-ийг үндэслэн дараах зүйлс шинэчлэгдлээ:

- **Өнгө**: cream `#fbf4e8`, hot pink `#ff6698`, deep green `#0f4b42`, blush `#efb8c3`,
  хар/цагаан — бүгд таны alban ёсны палеттаас.
- **Лого**: header, footer, favicon бүгдэд таны 2 PNG (icon mark + cursive wordmark)
  ашиглагдаж байна (`assets/images/`).
- **Фонт**: Танай бодит **Museo Sans Cyrillic** фонтын файлууд (`.ttf`) одоо WOFF2 хэлбэрт
  хөрвүүлэгдэж `assets/fonts/`-д орсон, бүх хуудас үүгээр харагдана (жинхэнэ brand фонт,
  орлуулагч биш). Боломжтой жинд: 100, 300, 500 (Regular), 700 (Bold) — энгийн ба Italic,
  мөн 900 Italic. 900 Regular ирээгүй тул маш зузаан текст хэрэгтэй газар 700-р орлуулсан.
- **Дугуйрсан хэлбэр** (squircle) — карт, товчнуудын radius ихэссэн, таны icon mark-тай
  адил "зөөлөн дугуй" харагдацтай болгосон.

Хэрэв ирээдүйд лого, өнгөнд өөрчлөлт орвол хэлээрэй — дахин тааруулж болно.

## 3. Netlify дээр байршуулах — АНХААРАХ: deploy арга өөрчлөгдсөн

Өмнө нь зүгээр drag & drop хийдэг байсан, гэхдээ **жинхэнэ нууцлалтай Админ хэсэг**
(Netlify Function) ажиллахын тулд Netlify-д "зүгээр файл чирэх" биш, доорх 2 аргын
аль нэгээр deploy хийх шаардлагатай (Function-ыг Drag & Drop дэмжихгүй):

### Хувилбар А — Netlify CLI (хамгийн хялбар, drag & drop-той ойролцоо)

1. Node.js суулгаагүй бол: https://nodejs.org → "LTS" хувилбарыг татаж суулгана
2. Терминал/Command Prompt нээгээд:
   ```
   npm install -g netlify-cli
   netlify login
   ```
   (Browser нээгдэж танай Netlify account-аар зөвшөөрнө)
3. Энэ `zt-site` фолдер дотор очно (`cd` командаар), дараа нь:
   ```
   netlify link
   ```
   → Жагсаалтаас өөрийн одоо байгаа сайтыг (drag & drop-оор үүсгэсэн) сонгоно.
   Хэрэв шинэ сайт хийх бол `netlify init` ашиглана.
4. Deploy хийнэ:
   ```
   netlify deploy --prod
   ```
   Энэ удаа function-той хамт бүгд орно.

### Хувилбар Б — GitHub-тэй холбох (автоматаар update хийгддэг болгох)

1. Энэ фолдерыг GitHub дээр шинэ repository болгож оруулна (GitHub Desktop эсвэл
   `git init && git add . && git commit -m "init" && git push` гэх мэт)
2. Netlify дашбоард → "Add new site" → "Import an existing project" → GitHub repo-гоо сонгоно
3. Publish directory: `.` (root), Functions directory: `netlify/functions` (netlify.toml-д
   аль хэдийн бичигдсэн тул автоматаар таних ёстой)
4. Цаашид зүгээр `git push` хийхэд л шинэчлэгдэнэ — drag & drop хэрэггүй болно

### Хоёр аргын дараа: Environment Variables тохируулах

Netlify дашбоард → таны сайт → **Site configuration → Environment variables → Add a variable**
дараах 3 утгыг нэг нэгээр нэмнэ (бүгд **"Sensitive"/нууц** төрлөөр):

| Key | Утга |
|---|---|
| `JSONBIN_BIN_ID` | jsonbin.io-д үүсгэсэн Bin-ийн ID |
| `JSONBIN_MASTER_KEY` | jsonbin.io → API Keys хуудасны Master Key |
| `ADMIN_PIN` | Админ хэсэгт нэвтрэх дуртай 4-8 оронтой код |

jsonbin.io тохиргоо (Bin үүсгэх):
1. https://jsonbin.io → нэвтрэх (Staff app-даа ашигладаг account-тойгоо адил байж болно)
2. **Bins** хуудас → **"Create Bin"** → агуулга: `{"orders": []}` → хадгал → Bin ID-г хуулна
3. Bin Settings-ээс **"Enable Versioning"** асаа (хэн нэгэн буруу бичвэл буцаан сэргээх боломжтой)
4. **API Keys** хуудас → Master Key-г хуулна (шинээр Access Key үүсгэх шаардлагагүй болсон —
   учир нь key одоо зөвхөн серверт байх тул бүрэн эрхтэй Master Key ашиглахад аюулгүй)

Environment Variables-ийг нэмсний дараа **дахин deploy хийнэ** (`netlify deploy --prod` эсвэл
git push) — шинэ утгууд function-д хүрэхийн тулд.

Админ хэсэг рүү орохдоо: `таны-сайт.netlify.app/admin.html`
(Энэ холбоосыг публик нэвтрэх цэс рүү зориудаар оруулаагүй — зөвхөн та мэднэ.)

## 4. Жинхэнэ нууцлал — юу өөрчлөгдсөн бэ?

Өмнөх хувилбарт Access Key, PIN хоёул browser-ийн эх кодод (`assets/config.js`)
харагдаж байсан. Одоо:

- ✅ `JSONBIN_MASTER_KEY`, `ADMIN_PIN` хоёул **зөвхөн Netlify-н серверт** (Environment
  Variables) байна — browser-т хэзээ ч очихгүй, "view source" хийсэн ч харагдахгүй.
- ✅ Админ хэсгийн **PIN бүрийг серверт шалгадаг** (`netlify/functions/orders.js`) —
  зөв PIN-гүйгээр захиалгын мэдээлэл уншигдахгүй.
- ✅ Захиалга бүртгэх нь нийтэд нээлттэй (PIN шаардахгүй, хэн ч захиалга өгч чадна),
  гэхдээ **захиалгуудыг харах/засах боломж зөвхөн зөв PIN-тэйд** олгогддог.
- ⚠️ PIN тань хэрэглэгчийн browser-ийн sessionStorage-д хадгалагдана (нэг tab нээлттэй
  байх хугацаанд, дараагийн нэвтрэлт бүрд дахин шалгагдана) — энэ хэвийн practice.

Хэрэв та ирээдүйд илүү өндөр түвшний хамгаалалт (жишээ: хэрэглэгч тус бүрт өөр нэвтрэх
мэдээлэл, нэвтрэлтийн хугацаа дуусдаг token, г.м.) хүсвэл хэлээрэй — нэмж хийж болно.

## 5. Бодит зураг

**CuteCup аяга (48), Бэлгийн багц (17), Цэцэгс (20) — нийт 85 бараа бодит зурагтай боллоо**
(zuvhuntuund.com/Zochil-ийн одоогийн зурагнаас шууд татсан). "Гол бэлгээ сонгоорой" хуудас
дээр одоо жинхэнэ зураг харагдана.

Зөвхөн **Дагалдах зүйлс, Сав** ангиллын ~18 жижиг зүйл (sticky note, лаа, дугтуй, хайрцаг
зэрэг) л placeholder icon хэвээр байна — эдгээр нь танай студиогийн шинэ зургаар солих
төлөвлөгөөтэй байсан тул зориудаар орхисон.

Бодит зураг (өөрийнхөөрөө studio зураг, эсвэл дээрхийг солих) нэмэхдээ:
1. Зургаа `assets/images/products/` фолдерт хийнэ (жишээ: `776017.jpg`)
2. `assets/data.js`-д тухайн барааны object-ын `"image"` талбарыг шинэ замаар бичиж дарна:
   `"image": "assets/images/products/776017.jpg"`
   (одоо бүх 85 бараа `"image": "https://cdn.zochil.shop/..."` гэсэн зурагтай — танай
   шинэ зургаар зүгээр дарж бичихэд бэлэн)

> Анхаарах зүйл: эдгээр зургууд яг одоохондоо Zochil-ийн серверт (`cdn.zochil.shop`) шууд
> хадгалагдаж байгаа тул хэрэв тэр зураг Zochil дээрээс устгагдвал энэ сайт дээр ч харагдахгүй
> болно. Тогтвортой байлгахын тулд ирээдүйд эдгээрийг `assets/images/products/`-руу татаж аваад
> бичих нь зүйтэй — хүсвэл хэлээрэй, дараа хийж өгье.

## 6. Бараа/үнэ шинэчлэх

Бараа нэмэх/хасах/үнэ засах бол `assets/data.js`-ийн `PRODUCTS` массивыг засна. Талбарууд:

```js
{ id, name, price, oldPrice (хямдралгүй бол null), category ("cup"|"giftset"|"flower"|"extra"|"box"),
  soldOut (true/false), url (Zochil-ийн бодит барааны холбоос) }
```

`recipients` талбар автоматаар тооцоологддог (`tagRecipients()` функц) — нэр/ангиллаас санаа авч
"Ээждээ", "Хайртдаа" гэх мэт санал болгодог. 100% зөв биш, heuristic учраас цаг гарвал гараар
тааруулж болно.

## 7. Захиалгын урсгал ба Админ хэсэг

Builder-ийн төгсгөлд хэрэглэгч нэр, утасны дугаараа үлдээж, дараа нь Алхам 6 дээр:
- Барааны дүн дээр **хүргэлтийн 7,000₮ автоматаар нэмэгдэж**, нийт төлөх дүн гарна
  (`builder.html`-ийн `DELIVERY_FEE` constant-аар хялбар өөрчилж болно)
- Захиалга бүрт **давтагдашгүй захиалгын дугаар** (жишээ: `ZT482913`) үүснэ
- Хүргэлтийн нөхцөл, төлбөрийн анхааруулга харуулж, **"Зөвшөөрч байна"** товч дарсны
  дараа дансны мэдээлэл (Эзэмшигч: Энхжин, Данс, Гүйлгээний утга = захиалгын дугаар) гарч ирнэ
- **"Захиалгыг бүртгэх"** товчоор — захиалга шууд танай Админ хэсэгт бичигдэнэ (дугаар,
  барааны дүн, хүргэлт, нийт дүн бүгдтэй)
- **"Хуулах"** эсвэл **Messenger/имэйл** товчоор — гар аргаар илгээх боломжтой хэвээр

Админ хэсэгт захиалга бүрийн **дугаарыг харж**, ирсэн дансны гүйлгээтэй (гүйлгээний утга)
шууд тулгаж шалгах боломжтой. Захиалга бүрийг **Шинэ → Баталгаажсан → Хийгдсэн** (эсвэл
Цуцалсан) гэж тэмдэглэж болно. Дээд талд төлөв тус бүрийн тоо харагдана.

## 8. Админ хэсгээс бараа нэмэх, засах (шинэ!)

`admin.html` дээр одоо **"Бараа удирдах"** таб нэмэгдсэн. Энд:
- **Шинэ бараа нэмж** болно (нэр, үнэ, хуучин үнэ, ангилал, зураг, дэлгүүрийн холбоос)
- **zuvhuntuund.com-оос татсан 103 бэлэн бараа** (CuteCup, Бэлгийн багц, Цэцэгс, г.м.)
  **БҮГДИЙГ ХАРАХ, ЗАСАХ** боломжтой боллоо — нэр хайх, ангиллаар шүүх боломжтой жагсаалт.
  Бараа бүрийн "Засах" товч дарвал: үнэ, хуучин үнэ, ангилал, Builder-ийн алхам (role),
  Дууссан төлөв, зураг/дэлгүүрийн холбоос бүгдийг өөрчилж "Хадгалах" дарна.
- Барааг **"Гол бэлэг"** (Алхам 3-т харагдах), **"Нэмэлт зүйлс"** (Алхам 4) эсвэл
  **"Сав, баглаа боодол"** (Алхам 2) гэж сонгоно — энэ нь builder-ийн алхам бүрт яг хаана
  харагдахыг тодорхойлно
- Шинээр нэмсэн барааг бүрэн **устгах** боломжтой; 103 үндсэн барааны хувьд засвар хийсний
  дараа **"Анхны байдалд оруулах"** товчоор хийсэн засвараа арилгаж, эх (Zochil) утга руу
  буцаах боломжтой

Засвар/нэмэлт бараа шууд **Бүтээгдэхүүн хуудас, Нүүр хуудас, Бэлэг бэлдэх builder** бүгдэд
харагдана (сайт ачаалах болгонд Function-оор автоматаар татагдаж нэгтгэгдэнэ).

(Техникийн тайлбар: 103 үндсэн барааны жинхэнэ өгөгдөл `assets/data.js`-д хэвээр байгаа бөгөөд
**хэзээ ч устгагдахгүй/өөрчлөгдөхгүй** — админ хийсэн засвар нь зөвхөн дээр нь "наалддаг"
давхарга (`overrides`) бөгөөд JSONBin bin-д таны захиалга, шинэ бараатай хамт
`{"orders":[...], "products":[...], "overrides":{...}}` хэлбэрээр хадгалагдана. Шинэ Bin
үүсгэх шаардлагагүй.)

## 9. Олон барааг зэрэг засах (Bulk edit) — шинэ!

103 барааг нэг бүрчлэн ангилах хэцүү байсан тул одоо **олныг зэрэг сонгож, нэг товчлуураар
шинэчлэх** боломжтой боллоо:

1. "Бараа удирдах" таб → хайх/ангиллаар шүүгээд хүссэн барааг олно
2. Бараа бүрийн зүүн талд байгаа **checkbox**-аар хэдийг ч хүссэн сонгоно
   (эсвэл **"Жагсаасан бүгдийг сонгох"**-ыг ашиглаад шүүлтүүртэй таарсан бүгдийг нэг дор сонгоно)
3. Дээд талд гарч ирэх **хар bulk-bar** дотроос Ангилал, Алхам (role), Төлвийг сонгоод
   **"Хэрэгжүүлэх"** дарна — зөвхөн сонгосон талбарууд л өөрчлөгдөнө, бусад нь хэвээр үлдэнэ

Жишээ хэрэглээ: "Цэцэгс" ангиллаар шүүгээд бүгдийг сонгож, нэг дор "Алхам" талбарыг
тохируулах, эсвэл нэрээр хайгаад олдсон 10 барааг зэрэг "Дууссан" болгох.

Асуух зүйл байвал хэлээрэй — дараагийн ярилцлагад үргэлжлүүлж болно.
